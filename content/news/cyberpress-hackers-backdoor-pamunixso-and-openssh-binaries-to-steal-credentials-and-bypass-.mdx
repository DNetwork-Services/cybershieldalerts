Threat actors linked to China have maintained a hidden presence inside a highly secure critical infrastructure network for nearly a decade. Tracked by Sygnia as Velvet Ant, the group orchestrated an elaborate campaign dubbed Operation Highland to infiltrate an environment with no direct internet access. Forensic artifacts from the incident reveal an undetected network presence dating back to 2016. Instead of relying on new software vulnerabilities, the attackers systematically compromised the core authentication stack by backdooring Linux PAM modules and OpenSSH binaries. This deep level of access allowed the group to quietly harvest credentials, monitor administrative commands, and completely bypass normal authentication flows while surviving routine password resets. Backdoored Binaries Steal Credentials The target network was deeply segregated, forcing Velvet Ant to build a deliberate, multi-stage access chain. They began by compromising internet-facing servers, deploying a modified version of GS-Netcat disguised as a legitimate system utility. This tool established a stealthy, encrypted reverse shell to a remote command-and-control server. To ensure persistent access across system reboots, the hackers planted malicious startup scripts and disguised systemd unit files as benign Chrome services. Snippet from IDA showing the usage of GS-Netcat (Source: sygnia) Alongside GS-Netcat, Velvet Ant utilized a custom SOCKS5 proxy script to facilitate lateral movement. This proxy enabled threat actors to tunnel network traffic through the compromised hosts without attracting attention. From there, they pivoted deeper into the IT network by abusing Nginx servers. By modifying Nginx configurations, they proxied incoming web requests through a FastCGI wrapper to execute custom binaries. This execution bridge acted as a gateway, ultimately allowing the attackers to establish SSH connections directly into the isolated critical infrastructure segment without ever needing a direct internet connection. Once inside the secure network, Velvet Ant shifted their focus to long-term survival by taking over the authentication layer itself. snippet of the unit file under ‘/lib/system/system’ showing the execution of ‘auditdb’ tool (Source: sygnia) They replaced legitimate pam_unix.so modules with custom-built malicious variants across multiple compromised machines. Investigators discovered nine distinct variants of these modules, each compiled in separate environments, highlighting a highly resourced operation. These backdoored PAM modules intercepted the authentication process to grant unauthorized access when a hardcoded backdoor password was entered. Furthermore, they covertly recorded legitimate usernames and passwords, saving the stolen data to hidden log files. The group also deployed heavily modified OpenSSH suites, injecting malicious capabilities into standard ssh, sshd, and scp binaries. These tampered files dramatically expanded the attackers&#8217; visibility and control over the compromised network infrastructure, Sygnia said. Remediating this intrusion proved exceptionally risky because the malware was embedded in the exact tools administrators needed to manage the network. Replacing core PAM and SSH components blindly could have resulted in a total administrative lockout or severe production outages. Responders had to build an isolated lab environment to test the eradication process, ensuring that the necessary security updates would not disrupt business continuity. Operation Highland serves as a stark reminder that isolated networks are not immune to compromise, emphasizing the need for proactive threat hunting beyond standard alert-driven security operations.
