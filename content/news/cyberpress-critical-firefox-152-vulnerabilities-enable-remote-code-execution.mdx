Mozilla released Firefox 152 on June 16, 2026, patching a sweeping set of 40 security vulnerabilities, including multiple high-severity flaws that could enable remote code execution, sandbox escapes, and privilege escalation affecting millions of users. The most critical vulnerabilities in this release involve memory corruption and use-after-free conditions across core browser components. CVE-2026-12291, reported by Zijie Zhao, exposes a use-after-free flaw in the Networking: HTTP component, while CVE-2026-12293, reported by superhei, affects the Graphics: WebGPU component through a similar memory mismanagement issue. Both carry high-severity ratings and could allow attackers to execute arbitrary code by exploiting freed memory regions during active browser sessions. Critical Firefox 152 Vulnerabilities Patched CVE-2026-12289, discovered by choeseyeong, targets the Graphics: WebRender component with a privilege escalation vector, potentially allowing a low-privileged attacker to gain elevated system access. Meanwhile, CVE-2026-12292 highlights incorrect boundary conditions in the Web Audio component a class of vulnerability often leveraged to trigger out-of-bounds memory access. Four high-severity sandbox escape vulnerabilities stand out as particularly dangerous, as they undermine Firefox&#8217;s core isolation model. CVE-2026-12294 and CVE-2026-12295, affecting the DOM: Workers and DOM: Navigation components, respectively, were both reported by Quy Pham and Yaqoub Aldurayhim. CVE-2026-12296, also by Aldurayhim, targets the Security: Process Sandboxing component directly. CVE-2026-12297, reported by researcher &#8220;zx,&#8221; exploits incorrect boundary conditions in the Networking component to escape the sandbox entirely. Successful exploitation of any of these flaws could allow malicious web content to escape the browser&#8217;s sandboxed environment and interact with the underlying operating system. CVE-2026-12326 and CVE-2026-12328 represent clusters of collective memory safety bugs identified by Mozilla&#8217;s fuzzing team and external researchers. Both carry high-severity ratings, with Mozilla explicitly noting that &#8220;some of these bugs showed evidence of memory corruption and we presume that with enough effort some of these could have been exploited to run arbitrary code.&#8221; CVE-2026-12299, reported by Hyeonjun Ahn, introduces a JIT miscompilation flaw in the DOM: Core & HTML component, a vector that has historically been used to bypass memory protections. Beyond the critical findings, Mozilla addressed 16 moderate-severity vulnerabilities, including memory safety bugs; mitigation bypasses in the DOM: Security component (CVE-2026-12302, CVE-2026-12315, CVE-2026-12316). Same-origin policy bypass via Networking: Cookies (CVE-2026-12304) and information disclosure with sandbox escape in Process Sandboxing (CVE-2026-12311, CVE-2026-12313). Lower-severity issues include a Password Manager information disclosure (CVE-2026-12320), a clickjacking flaw in the GTK widget component (CVE-2026-12322), and a denial-of-service in the Audio/Video Playback component (CVE-2026-12319). According to Mozilla advisory, all Firefox users are urged to update to Firefox 152 immediately. Users on extended support releases should upgrade to Firefox ESR 140.12 or Firefox ESR 115.37. Organizations deploying Firefox in enterprise environments should prioritize patch rollout given the sandbox escape and RCE potential demonstrated in this advisory.
