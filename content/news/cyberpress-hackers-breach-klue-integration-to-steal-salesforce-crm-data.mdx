Threat actors exploited a compromised Klue Battlecards integration to silently harvest Salesforce CRM data from enterprise environments in June 2026, leveraging OAuth tokens and automated Python scripts to pull bulk records through legitimate API channels, all while flying under the radar of conventional security monitoring. ReliaQuest researchers observed adversaries authenticating through compromised Klue integration service accounts, Klue being a competitive-intelligence platform that syncs battlecard and win/loss data with Salesforce, generating OAuth tokens, and deploying automated Python scripts identifiable by their Python-urllib user-agent strings. The attack chain began with object catalog enumeration via GET /services/data/v59.0/sobjects, followed by looped REST API queries against Salesforce&#8217;s query endpoint (/services/data/v59.0/query), with results paginated through the QueryMore cursor for nearly 24 continuous hours. Hackers Breach Klue Integration The exfiltration occurred in two distinct phases. The first was a slow, sustained pull designed to blend in with legitimate integration traffic. The second shifted to a concentrated burst of nearly 1,000 queries within a 15-minute window, suggesting either time pressure or a pivot to targeted high-value records. In a separate environment, extraction was maintained for a 6-hour period. Salesforce has since disabled the Klue Battlecards app connection across all affected customer environments as a containment measure, confirming that the issue originates from the integration layer rather than any core platform vulnerability. The attack closely mirrors a series of third-party OAuth-abuse campaigns that targeted Salesforce ecosystems throughout 2025 and 2026. In June 2025, threat group ShinyHunters used voice phishing to trick employees into authorizing malicious connected apps before bulk-extracting Salesforce data for extortion. In August 2025, the cluster tracked as UNC6395 stole OAuth refresh tokens from the Salesloft Drift integration, querying Salesforce across hundreds of organizations, the closest known analog to the Klue incident. The defining pattern across all three incidents is the same: abuse of OAuth tokens or credentials from a trusted third-party SaaS vendor, exploiting the fact that integration service accounts carry persistent, often broadly scoped API access yet receive far less monitoring scrutiny than employee accounts. While the bulk CRM theft methodology and Python-based tooling overlap with ShinyHunters&#8217; known tradecraft, researchers note key differences. UNC6395-linked activity used user agents, including python-requests and Salesforce-Multi-Org-Fetcher and frequently routed traffic through Tor, whereas the Klue incident used a generic Python-urllib string with data-center hosting. No extortion demand or leak-site posting has been observed. Attribution to ShinyHunters, UNC6395, or an entirely separate operator remains open. This incident does not involve a CVE-assigned vulnerability. The attacker leveraged valid or stolen OAuth credentials via a pre-authorized integration; no exploit was needed because the door was already open. Mitigation Revoke and rotate all credentials tied to the Klue integration, including refresh tokens, client secrets, and active OAuth grants. Password resets alone do not sever persistent token-based access Audit Salesforce REST API logs for unusual query volumes, repeated QueryMore pagination, Python-urllib user-agent strings, and access from unrecognized IP addresses Enforce IP allowlisting on all third-party connected app accounts and apply the same controls to SIEM and SOAR API endpoints Inventory all OAuth-connected integrations and scope them to least-privilege access, treating them with the same rigor applied to privileged human accounts ReliaQuest assesses that it is highly likely that threat actors will continue targeting third-party Salesforce-connected integrations through the remainder of 2026, as the OAuth-abuse playbook is repeatable, effective, and now widely adopted across multiple threat actor ecosystems.
