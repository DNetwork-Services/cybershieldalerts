Microsoft has announced it will discontinue updates to the OneDrive sync app for devices running Windows 10 version 21H2 and earlier, effective August 15, 2026. The change, detailed in Message Center post MC1426708, means affected machines will stop receiving feature updates, bug fixes, and critical security patches for the sync client, leaving organizations with a narrowing window to act. The update tightens the alignment between OneDrive&#8217;s support lifecycle and the underlying Windows 10 servicing timeline. Devices running Windows 10 version 22H2 will continue receiving full sync app support, including security updates, through October 10, 2028. Microsoft Ends OneDrive Sync App Security Updates Devices on version 21H2 and earlier, however, will lose all sync app updates starting August 15, 2026. Existing installations may continue running afterward, but Microsoft offers no guarantee of continued functionality. This is a retirement-classified &#8220;Major change&#8221; per the Message Center tagging, first published July 13, 2026, and updated July 17, 2026, with revised content. For unpatched sync clients, the risk isn&#8217;t hypothetical. OneDrive sync apps handle continuous authentication tokens, file metadata, and local caching of enterprise data, all attractive targets if vulnerabilities go unaddressed. Once security updates stop, any newly discovered flaw in the sync client on these older builds becomes a permanent, unpatched attack surface, comparable to running an EOL browser or unsupported OS component with active network connectivity to Microsoft 365 tenant data. Organizations that haven&#8217;t moved off 21H2 or earlier are already behind, since Windows 10 21H2 itself exited mainstream servicing well before this OneDrive-specific cutoff. Microsoft&#8217;s announcement essentially closes a lingering gap where devices could still receive at least sync app patches despite lagging on broader OS updates. Sync functionality may degrade gradually rather than fail outright, making detection harder for IT teams relying on user-reported issues rather than proactive monitoring. No tenant-side configuration changes are required, so this is purely a device-level exposure issue rather than something admins need to reconfigure centrally. OneDrive web access remains fully supported via browser, giving users a fallback path for file access without the vulnerable local client. Microsoft has not flagged any explicit compliance considerations, though organizations subject to regulatory frameworks requiring patched endpoints should treat this as a de facto compliance gap worth reviewing internally. Security and IT teams should treat this as a lifecycle-driven risk reduction task. Start by inventorying devices still running Windows 10 21H2 or earlier across the fleet, then prioritize upgrades to Windows 11 where hardware permits, falling back to Windows 10 22H2 as the minimum viable target where a full upgrade isn&#8217;t feasible. Notify end users and help desk staff well ahead of the August 15 cutoff to reduce support ticket volume once the change takes effect. Point affected users toward OneDrive on the web as an interim access method during migration, and update internal documentation and asset management records to reflect the new support boundaries. Microsoft has not indicated any extension or paid support option for the sync app on these older builds, reinforcing that OS-level upgrades remain the only durable fix.
